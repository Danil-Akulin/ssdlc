Waterfall
Waterfall, or the waterfall model, is a traditional methodology that has been around since the 1970s. In it, the development of the project is divided into stages: from the analysis of system requirements to the release of the product. Each step is a separate phase of software development, and the team must complete one phase before moving on to the next.
Can be used in aircraft, military or space industries


Iterative
Iterative software development is the process of creating software, which is carried out in small stages, during which the analysis of the obtained intermediate results is carried out, new requirements are put forward and previous stages of work are corrected.
Can be used in marketing, education


Prototyping
The prototype model is used to develop an incomplete program and its versions. It is a software development method. It is used to build and test an earlier version of the final product. This is an iterative process, and this model mostly works in situations where the requirements are available in advance. In the prototype model, the requirement should be defined in as much detail as possible. A design is developed for a new system and the first prototype is created from this design.
Can be used in marketing, education


Spiral
The spiral model represents a software development process pattern that combines the ideas of the iterative and waterfall models. ... In the spiral model, the life path of a product being developed is depicted as a spiral, which, starting at the planning stage, unwinds with the passage of each next step.
Can be used in marketing, education


V - shape
The V-Model (or VEE model) is an information systems (IS) development model aimed at simplifying the understanding of the complexities associated with systems development. It is used to define a unified procedure for the development of software products, hardware and human-machine interfaces.
Can be used in marketing, education


Agile
Agile is an iterative development model in which software is built incrementally from the very beginning of a project, as opposed to waterfall models where code is delivered at the end of a work cycle. The basis of an agile methodology is breaking projects into small working pieces called user stories.
Can be used in marketing, education

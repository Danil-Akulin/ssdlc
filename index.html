Design driven development
Domain driven design (DDD) is an approach to application development based on the allocation of domains (domain). Domain is the area of knowledge/activity for which the application is being developed.


Domain driven design
DDD is an approach that is aimed at studying the subject area of the enterprise as a whole or some individual business processes.
This is a great approach for projects where the complexity (entanglement) of the business logic is quite high. Its use is intended to reduce this complexity as much as possible.


Secure by design
Secure by design (SBD) in software engineering means that software is designed from the start to be secure.
In this approach, alternative security tactics and patterns are considered first; among them, the best are selected and brought to life by architectural design,
and then used as guidelines for developers Wikipedia site: hrwiki.ru


Test - driven development
it is a software development methodology that is based on the repetition of short development cycles: initially a test is written,
covering the desired change, then code is written that implements the desired behavior of the system and allows the written test to pass,
and then refactoring of the written code is carried out with a constant check of the passage of all tests.


Acceptance test driven development
The general idea is that before you do something, you need to come up with a criterion for the work done and a criterion for the fact that the work is done correctly.
Why is it important? Because these criteria allow you to understand at a very early stage what exactly needs to be done, how to do it, what exactly to count
good result

Data driven development
is a programming method or even a programming paradigm in which the program code, although separated from the input data,
but is designed in such a way that the logic of the program is determined by the input data. In a data-driven program, a part or even
all its properties are set at runtime, which is especially important if the program is written by the user or must be used by the user
change without recompilation.
